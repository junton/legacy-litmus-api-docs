We now know we have two Client objects stored in emailTest.Results. A Client object has a lot of properties, all filled with useful information when displaying the result for that client to your users. We'll cover those below for our first result, retreived in pseudo code below:

 

var result = emailTest.Results[0]

 

Remember that we have two results to check, but we'll focus on the first for now:

 

ApplicationLongName

Type:          string

Capacity:     255 characters

Value:         "Outlook 2003"

 

The longer, friendlier name of the client that you can show to your users. 

 

ApplicationName

Type:          string

Capacity:     255 characters

Value:         "OL2003"

 

A unique identifier for each client, we guarantee that this name won't change for any given client (although we may deprecate one and announce another in its place, eg "IPHONE2" may be replaced by "IPHONE3", with an overlap inbetween, you will be notified of this). 

 

AverageTimeToProcess

Type:          integer

Capacity:     4 (32-bit integer)

Value:         125

 

Litmus constantly keeps an eye on every client and updates this property to indicate the average amount of time (in seconds) it is currently taking to process a result in this client. You may chose to show this value to your users while they're waiting for a test.ff

 

To find the total time until a test will complete, select the highest value of all clients' AverageTimeToProcess. For example, if you perform a test on two clients and one returns 125, the other 180, the total time to process is 180 seconds (or 3 minutes). 

 

BusinessOrPopular

Type:          boolean 

Capacity:     1 

Value:         true / 1

 

If Litmus currently considers this client a "business client", this value will be true. Business clients include Outlook and Lotus Notes.

 

Completed

Type:          boolean 

Capacity:     1 

Value:         true / 1

 

This is an important property as, when true, it indicates that the result has finished processing and the result is ready to be viewed. Please note that if a result fails for some reason, Completed is always set to true. It is therefore safe to use the Completed property to know when to stop polling for updates for a particular result, and when to show the result to your user.

 

DesktopClient

Type:          boolean 

Capacity:     1 

Value:         true / 1

 

Desktop clients are those that run locally, on the desktop. Examples include Outlook, Lotus Notes, Apple Mail and Thunderbird. Email clients such as Gmail, AOL and Hotmail would have a DesktopClient value of false.  

 

FoundInSpam

Type:          boolean 

Capacity:     1 

Value:         false / 0

 

Indicates if the email was found in this client's spam folder. Since not all clients support this property, it may always be false for some cilents.  

 

PlatformLongName 

Type:          string

Capacity:     255 characters

Value:         "Microsoft Windows"

 

The long, friendly name of the platform this client is running on. In this case of multi-platform applications (such as Thunderbird), the platform Litmus uses is used. The PlatformLongName includes the platform's manufacturer, as well as the platform name.

 

In the case of web-based email clients, both PlatformLongName is set to "Web-based", if you wish to customise this, use the DesktopClient boolean property to detect if the client has an operating system in its platform name and act accordingly.

 

PlatformName  

Type:          string 

Capacity:     255 characters 

Value:         "Windows"

 

The more detailed, name of the platform the client is running in.

 

In the case of web-based email clients, we will return Firefox, Chrome or Explorer indicating the specific web browser being used. 

 

RenderedHtmlUrl 

Type:          string 

Capacity:     255 characters

Value:         "" / string.Empty / null / nil / nothing

 

Reserved. Please ignore.

 

ResultType 

Type:          string 

Capacity:     10 characters

Value:         "email"

 

Contains either "email", "spam" or "page". If you've selected any spam clients for your email test, you will have some results with a ResultType of "spam". Most resellers chose to display spam results in a separate section. 

 

SpamScore

Type:          double precision floating point number 

Capacity:     8 (64-bit) (you should use a type capable of storing double or float. Storing as an integer (64-bit or otherwise) will truncate the value)

Value:         "" / string.Empty / null / nil / nothing

 

If ResultType was equal to "spam", this property may contain a score left by the spam filter this Client object represents. Understanding spam results is covered in greater detail later.

 

Id

Type:          integer

Capacity:     4 (32-bit)

Value:         3680253

 

Unlike EmailTest.ID, Client.Id is correctly named and is of type integer instead of string. In future versions of the Litmus Web Service, EmailTest.ID will be renamed EmailTest.Id and be converted to an integer. The WSDL you are using now will remain as it is.

 

This property represents a unique identifier for this result (not this client). You can use this id when using GetResult, this is covered in more detail later.

 

State

Type:          string 

Capacity:     15

Value:         "complete"

 

State is the current state of your result, it can be either "pending", "complete" or "error":

 

State Description
pending The result is still processing. You should poll for updates again in 60 seconds time.
complete The result has completed. You should now stop polling for updates.
error The result has failed. You should now stop polling for updates.

 

The property State is not to be confused with Status (see below).

 

Status

Type:          integer 

Capacity:     2 (16-bit or short)

Value:         0

 

Represents a client's current status. A status of 0 indicates all is well, other values are:

 

Status Description
0 Client is performing well, no delays.
1 Client is running slower than usual, expect delays of up to 15 minutes.
2 Client is currently unavailable, you should avoid ordering new tests for this client, but any ordered tests will be honored when the client recovers.

 

Litmus uses this information to generate its status page, listing all clients currently experiencing delays or unavailability, seen in the screenshot below. You are free to use it however you think is best.

 

 

 

The property Status is not to be confused with State (see above).

 

SupportsContentBlocking

Type:          boolean

Capacity:     1

Value:         true / 1

 

If a client supports both "images on" and "images off" captures, that is, the client has some kind of view where the user can't see the images in their email, this property will be true.

 

Some clients, such as Apple Mail and some webmail clients, don't support external content blocking, and SupportsContentBlocking will be false.
