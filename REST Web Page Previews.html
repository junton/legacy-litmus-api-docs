Introduction

 

For Web Page Previews the work flow is:

 

  • Create an PageTest with Litmus and get a result back from us
  • Poll the api periodically to see if your previews are complete  

 

Details

 

Step 1 :: Start a Litmus web page test by sending a POST request to https://previews-api.litmus.com/api/v1/PageTests

 

You can customize the request by only requesting a few TestingApplications.

You can get a complete list of TestingApplications by making a GET request to https://previews-api.litmus.com/api/v1/PageTests/TestingApplications.  If you want to test all TestingApplications you can just send an empty PageTest object with only details about the url to be tested.

 

An example of a creating an web page preview for all TestingApplications would look like this:

 

curl -u username:password -X POST -H "Content-type: application/json"  https://previews-api.litmus.com/api/v1/PageTests -d '{ "Url" : "https://www.google.com/?", "Title" : "Google" }'

Step 2 :: Getting your results

 

You can check on the status of your results by making a GET request to https://previews-api.litmus.com/api/v1/PageTests/{id} using the Id you received in the response to Step 1.

 

Make sure to check the State of each TestingApplication.  A State of Complete means you can parse that result and the images.
